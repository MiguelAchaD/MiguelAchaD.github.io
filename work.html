Why the topic is important for networking

IPsec, meaning Internet Protocol security, is a set of protocols that provide security services for IP packets at the network layer. IPsec is widely used to communicate securely over the internet or any IP-based network and it is of great importance for networking for various reasons.
Firstly, it provides encryption for data transmission making sure that sensitive information remains private and inaccessible to others. It also ensures that data is unchanged and unaltered during transmission using mechanisms such as hashing.
Secondly, IPsec enables identity authentication of communication partners, this helps prevent unauthorized access and impersonation.
Thirdly, IPsec is a great tool to secure remote access to network resources through VPNs. In addition, it also facilitates communication security between geographically dispersed networks enabling secure connection over public networks.
All in all, IPsec is key for enhancing security, reliability and privacy of network communication in modern network environments.

How does it work / Foundations?

Once IPSec has been briefly presented, we will now focus on the bases and its functionality. For the purpose of understanding how it works, it should be remarked that the computers exchange data through this protocol thanks to the following steps.
However, before entering on how it works, some key points will be mentioned with the aim of facilitating the understanding of this complex procedure. Indeed, IPsec connections cover different technical aspects, which are the following ones.

  • Key exchange: Generally keys are necessary for encryption and this is not an exception. In this case, it is a random character string whose purpose is locking and unlocking messages. Furthermore, IPSec establishes keys, as well as a key exchange between the connected devices, in order to make each device able to decrypt the other one message.
  • Packet headers and trailer: Every data which is sent through a network is divided into smaller pieces called packets. Each packet contains both real information which is sent and headers about that data in order to make the computers understand what they need to do with the received packets. In the IPSec case, some headers are added into the packets which contain information about authentication or encryption. In addition, trailers, which go after each packet loading, are added.
  • Authentication: IPSec provides authentication for every packet, which acts as an authenticity mark. Thanks to this, it is ensured that the origin font is trustworthy and reliable.
  • Encryption: IPSec encrypts the util charge inside each packet as well as each packet header, although if a certain transport mode is used this could change. In other words, this makes sure that the data sent through IPSec is maintained secure and private.
  • Transmission: The encrypted packages travel one or more networks in order to achieve their destination. In this phase, the IPSec traffic has a significant difference from the IP normal one as the UDP protocol is used instead of the TCP one. The main reason why UDP is used is due to the fact that it allows the packets to pass through firewalls.
  • Decryption: On the other side of the communication, packets are decrypted and the applications, such as the browser, can now use the delivered data.

How is it applied and used in real-life situations?

As mentioned before, these protocols are useful for security between the communication outside and inside of a network. Taking this into consideration, really every activity done regarding the transmission of sensitive information between hosts can be benefited using these special protocols.

Common applications

Some of the most common applications are: VPNs, Site-to-Site Communication, Remote Access, Secure Web Communication, VoIP Security, IoT Security, Cloud Security, Mobile Device Security.


  • VPN: IPSec protocols are used on VPNs to provide secure and encrypted connections of users to a network. This is often used for corporations for their workers to complete tasks from a remote location, without risking sensitive data leaking.
  • Site-to-Site Communication: Same happens to the communication between networks as with VPNs, or provider services (such as cloud services) and a network. Helping to create secure connections with integrity.
  • Secure Web Communication: Web to client information exchange, such as banks and e-commerce sites, are widely used with the help of other protocols to reinforce the security of transactions. This application is key to ensure a strong service for clients.
  • VoIP Security: VoIP communication are the voice data packages transmitted over or in the network, its use has increased throughout these last years because of online meetings. The sensitivity of these is great, so to ensure a strong communication between ends, these protocols are also used.
  • Beneficiaries of the technology: Throughout this chapter it is clear that this technology benefits the user, and the party that applies this methodologies is the company that gives services. To implement this a set of policies and components have to be set, with the help of a cryptographic key`s management protocol this is possible. As a result all IoT devices can communicate free and securely with each other. Not risking any data leaks.
